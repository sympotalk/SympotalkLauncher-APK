# Incident Reports

장애·롤백·핫픽스 기록. 같은 실수를 반복하지 않기 위한 학습 자료.

## 파일명 규칙

```
YYYY-MM-DD-HHMM-<short-slug>.md
```

예: `2026-04-21-1530-apk-signing-mismatch.md`

## 템플릿

```markdown
# Incident: <제목>

- **발생**: 2026-04-21 15:30 KST
- **감지**: 2026-04-21 15:35 (GitHub Actions 실패 / 사용자 보고 / 모니터링)
- **해결**: 2026-04-21 16:10
- **심각도**: P0 (서비스 중단) / P1 (기능 일부 차단) / P2 (UX 저하)
- **영향 범위**: 누가 / 얼마나 영향받았는지

## 타임라인
- 15:30 — push to main (커밋 abc1234)
- 15:35 — APK 빌드 실패: "release.keystore 서명 불일치"
- 15:50 — 원인 식별
- 16:10 — 핫픽스 머지

## 근본 원인
(왜 이런 일이 일어났나)

## 즉시 조치
(어떻게 멈췄나)

## 수정
(영구 해결을 위한 변경)

## 재발 방지
- [ ] CI 게이트 추가
- [ ] 문서 업데이트
- [ ] 코드 리팩터링 이슈 등록 (docs/ISSUES.md)

## 관련 링크
- 커밋: <hash>
- 워크플로우 run: <URL>
- GitHub Issue: #<number>
```

## 자동화

`/rollback` 슬래시 커맨드가 이 디렉토리에 자동으로 기록합니다.
